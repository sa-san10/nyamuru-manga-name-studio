本プロンプトでは、ストーリー原稿を **Nyamuru Data Model（NDM）v10** に従うカラー漫画ネームへ構造化し、**Open Manga Name YAML（OMNY）形式** で出力します。

**Nyamuru Data Model（NDM）** は、漫画ネームの構造、コマ割り、人物、フキダシ、素材参照を定義するデータモデルです。愛称は **Nyamuru🐱** です。
**Open Manga Name YAML（OMNY）** は、NDMをYAMLとして保存・交換するためのシリアライズ形式です。

新しい原稿が渡されたら、下記の手順とテンプレートに従ってOMNY形式を出力してください。
出力するOMNYには、各ページのコマ割り座標（A4基準）と、画像生成AI（ChatGPT等）向けの座標系の説明文が必ず含まれます。

---

## 手順

1. **原稿を読み、ページ／コマ（パネル）単位に分解する。**
   原稿に「◯ページ目」の区切りがあればそれに従う。区切りが無い場合は内容のまとまりで判断し、分け方を一言添える。

2. **各ページのコマ割りを設計する。**
   - キャンバスはA4（210×297mm、縦長）を基準とする。
   - 🚨 **1ページのコマ数は最大6コマ（厳守）。** 7コマ以上入れるとセリフの描画が崩れる。コマを減らすのは自由（1〜6コマ）、増やすのは6コマまで。
   - 各コマは `rect`（四角）または `polygon`（多角形／対角線カット等）で指定する。
   - `id` は右上→左下の読み順で1から振る（日本漫画の右綴じ順）。
   - 内容に応じた強弱をつける：クライマックスは大コマ、テンポの速い場面は帯状の小コマ、感情の間は正方形に近いコマ、動きが激しい場面は対角線カットのポリゴン、など。
   - 🚨 **ページ間でコマ割りパターンを変えること（最重要）。** 全ページが同じ骨格（例：大トップ→右→左→大ボトム）になるのは禁止。ページごとに §D の中から**その場面に適したパターンを選ぶ**か、迷ったら**ランダムに違うものを選んで**変化をつける。隣り合うページで同じパターンを使わないのが原則。
   - **印象的なシーン（クライマックス・感動・転換点・登場の見得切り）では、パターン集に縛られず自由にネームを切ってよい。** 対角線カット、ページ2/3を使う大ゴマ、縦長コマ、細かい畳みかけからのドン、など漫画的な演出を優先する。
   - **キメ台詞・決意表明・啖呵・重要リアクション等の見せ場**は、話者の顔をアップにした大きめのコマを充てる。該当コマの `action` に「顔のアップ」「バストアップ」等を明記する。
   - 🪝 **ページ最終コマ（左下）は「ヒキ」として設計する。** 疑問・気配・動作の途中・「？」で終わるセリフなど、続きが気になる状態でページを切る。そして**次ページの冒頭コマ（大きめ）でその答えを受ける**（＝メクリ）。複数ページ原稿では、可能ならヒキになるシーンを先に決めてからページを割る。ヒキのコマは小さめ・次ページのメクリは大きめにすると緩急がつく（§DのパターンJ→Lが定石の組み合わせ）。
   - **フキダシが3個以上入るコマは、帯（横190mm級）以上の面積を確保する。** セリフの多いコマは大きく、セリフのないコマは小さくてよい——文字量とコマ面積を連動させる（文字は作画同時生成のため、面積不足は文字潰れに直結する）。
   - 🚫 **「田の字」禁止。** 2×2配置で上下の段の縦の溝（列の区切り線）が同じ位置に揃うと、読み順が「右→左」なのか「右列を縦に」なのか曖昧になる。2×2に割るときは段ごとに縦線の位置をずらす。
   - ✂️ **タチキリ（断ち切り）を見せ場に限り許可する。** 左右は `x=0` / `x=210`、下端は `y=297` まで紙端いっぱいにコマを拡張してよい（**上端はヘッダー帯があるため `y=10` まで**）。キャラ登場・見得切り・スプラッシュ等で画面の抜けを最大化したいコマに使う。多用すると効かなくなるため**1ページ1〜2コマまで**。タチキリ入りの定型は §D のパターンM〜Qを参照。
   - 各コマの外周に約2〜3mmの白マージン、コマ枠線は約1mm幅の黒を想定して座標を組む（**タチキリコマの断ち切り辺はマージンなしで紙端まで**。枠線は断ち切り辺には引かない）。
   - 参考パターンは **§D コマ割りパターン集** を参照（シーン別の使い分け目安つき）。

3. **各コマに人物配置（figures）とフキダシ配置（bbox/anchor）を指定する。**
   - `figures` にそのコマへ描くキャラを列挙し、各々に `bbox`（キャンバス絶対座標mm）・`anchor`（相対位置語彙）・`size`（full/waist-up/bust-up/face）を付ける。コマ外の声のキャラはfiguresに入れない。
   - 各フキダシに `bbox` と `anchor` を付ける。anchorと配列順（右上→左下）を一致させる。
   - 📐 **フキダシbboxの縦横比は、文字の組み方向に合わせる。** 縦書きのフキダシは**縦長**のバウンディングボックス（通常サイズの目安: w40 × h50mm）、横書きのフキダシは**横長**のバウンディングボックス（通常サイズの目安: w50 × h40mm）にする。文字量やコマの大きさに応じた拡縮は自由だが、縦長・横長の向きは崩さない。
   - 🚨 **フキダシbboxは、全figureの顔ゾーン（bbox上部1/3）と交差禁止。胴体との重なりは許容。** OMNY出力後、交差判定（x範囲重複∧y範囲重複）を全ペア自己検証し、結果を報告する。
   - bboxはコマ内に収める（polygonコマは斜辺の内側）。発言順・anchor・bboxの三者が同じ配置を指すよう揃える。

4. **各コマに背景詳細度 `bg`（0/1/2）を必ず指定する。**
   - **bg: 0 = 具体物なし・感情色ベタ＋効果**。風景や室内などの具体的背景は一切描かず、**そのシーン・感情に合った背景カラー（ベタ塗り or グラデーション）を敷いた上に**効果（集中線・漫符・トーン・オーラ・キラキラ・ハート等）を乗せる。**白一色の背景は禁止**（例：怒り=赤系、喜び=暖色系、驚き=黄系、緊張=紫〜紺系、感動=金〜桃系、静寂=青系）。**顔アップ・感情の爆発・キメ台詞・リアクション・アクションの瞬間**に使う。
   - **bg: 1 = 簡易背景**。**シーンに合った背景色グラデーション**の上に、その場を示す記号的な要素を1〜2点（壁のライン、窓枠、机の縁、ソファの背もたれ、手前の小物のシルエット等）。**白背景は避ける**。「手前のオブジェクト＋奥は色グラデ」の奥行き構成も良い。**通常の会話コマ・静かなシーン**のデフォルト。描き込まないが、色で空気はつくる。
   - **bg: 2 = 詳細背景**。ロケーションをしっかり描き込む「見せゴマ」。**1ページに2コマまで**（導入、場面転換、風景で魅せるコマ、スプラッシュ等）。どのコマを bg:2 にするかは固定ルールではなく、**ネーム担当が「見せたいコマ」に自由に設定する**（1コマ目が必ず2、ではない）。
   - 配分の目安：会話の応酬＝1、感情・キメ・オチ＝0、見せたいコマ＝2（位置は自由。ページ頭でも中盤の風景でもラストの余韻でもよい）。**1ページ内に0と1と2が混ざっているのが良い状態**。全コマ2は禁止（情報過多になり、視線誘導が弱くなる）。
   - `bg: 0` と「顔のアップ」「emphasis」は相性が良い——キメコマは基本 `bg: 0` にする。
   - bg:0/1 の背景カラーは、可能なら `action` に色の意図を添える（例：「背景は緊張の紫グラデ」）。指定がなければ作画AIがシーンから選ぶ。
   - **背景色の彩度は控えめが基本**。感情色のベタ/グラデは「雰囲気を伝える」ための色であり、キャラとセリフより主張してはいけない。**低〜中彩度（くすみ・パステル寄り）に抑える**のがデフォルト。**高彩度の鮮やかな背景は、感情の爆発・キメの最大強調など明確な演出意図があるコマだけ**に限定する（シーンに応じて彩度を調整し、理由なく原色ベタを敷かない）。

   **🎬 迷ったら映像演出をヒントにする**
   映画・アニメの映像文法は、そのままこのプロンプトの設計ヒントになる：
   - **背景色に困ったら「色のドラマカーブ（カラースクリプト）」**：ページ／作品全体の感情の起伏を、先に「色の変化の列」として設計する（例：日常の水色→違和感の黄→緊張の紫紺→静寂の深青→歓喜の金→余韻の橙）。各コマの色はそのカーブから拾えば迷わない。**隣接コマの色相差が大きいほど、場面転換・感情転換が強く伝わる**（じわじわ変える=時間経過、急に変える=事件）。
   - **bgの深さは「ショットサイズ」で考える**：bg2=引きのエスタブリッシング・ショット（ここはどこか、を1カットで示す）／bg1=ミディアムショット（会話の距離感）／bg0=クローズアップ（感情に寄る）。映画が「引き→寄り」で感情に近づいていくリズムを、bg 2→1→0の流れで再現できる。
   - **漫符・効果は「照明とレンズ」で考える**：逆光シルエット=シリアス・決意／トップライト（上からの光）=神聖・再会／下からの照明=不穏・恐怖／レンズフレアや光のボケ玉=感動・回想／被写界深度（手前ボケ）=没入。映像の照明・レンズ表現を、漫符やグラデーションの選択に翻訳するとコマの説得力が上がる。

5. **素材一覧（materials）を作り、コマとの対応を書く。**
   - この作品の作画に必要な**人物・背景・小道具の素材**を、トップレベルの `materials` に列挙する。
   - 各素材には、作画エージェントが**ファイル名を手がかりに素材を収集できるよう** `keywords`（検索キーワードの配列）を付ける（例: key「キャラAの普段着」→ keywords ["キャラA", "普段着", "私服", "立ち絵"]／key「キッチンの背景」→ keywords ["キッチン", "台所", "背景"]）。
   - `note` には見た目の要点を書く。**素材が見つからなかったときのデザイン指針**を兼ねる（例:「銀髪ロング・エプロン・そばかす」）。
   - 各コマの `assets` に、そのコマで使う素材の `key` を配列で書く（例: `assets: ["キャラAの普段着", "キッチンの背景"]`）。素材を使わないコマ（bg:0の感情コマ等）は省略してよい。
   - 作画エージェントは**作業前に** materials の keywords で手元の素材フォルダを検索・収集し、コマの assets に従ってリファレンスとして使う。**対応する素材が見つからない場合は、素材待ちで止まらず、noteと作品のスタイルに合わせて自由にデザインしてよい**（一度デザインした見た目は以後のコマでも維持する）。
   - ※ このプロンプトの責務は**必要素材の列挙（materials）とネーム内での素材指定（assets）まで**。素材アセットの収集・新規生成・検収・命名・フォルダ整理といった**素材管理の運用ルールは制作パイプライン側（manga-pipelineスキル等）が受け持つ**。

6. **文字は作画と同時に生成させる。後から写植する運用は禁止。**
   - セリフ・フキダシ・モノローグ・作品タイトル・ページ番号は、**画像生成AIが作画と同時に描画する**。各ページは「文字入りの完成漫画画像」として一度に生成される前提でOMNYを組む。
   - 「文字のない原稿を生成して、後からプログラム・画像編集・ローカル処理で写植する」ワークフローは**禁止**（フキダシの形・サイズと文字量の整合が取れず破綻する）。
   - 生成結果に文字化け・誤字・欠落・はみ出しがあった場合は、後加工で直さず**そのページだけ画像生成で再生成**する。
   - 生成された完成PNGは加工せず、そのまま納品する。

7. **セリフは必ず `bubbles`（配列）に入れる。**（セリフをコマ直下の個別キーで書いてはならない）
   - 一つのコマに複数のフキダシを入れてよい。連続する同一話者のセリフは、意味のまとまりごとに一つのコマへ寄せる。
   - **話者は各フキダシの `speaker` で示す。** 同じコマ内で直前のフキダシと同じ話者なら省略してよい。ナレーション（caption）・効果音（handwritten）など話者のいないフキダシも省略する。1コマに複数の話者がいる掛け合いもこれで表現できる。
   - **コマ内の発言順は、そのままキャラの立ち位置になる**（作画時、最初の話者がコマの右側・以降は発言順に左へ並ぶ——style_notes参照）。立ち位置を動かしたいときは発言順を組み替えるか、`action` に構図を明示する。
   - **フキダシには尻尾（しっぽ）を付けて話者を指す。** しっぽの先端は speaker のキャラの口元近くへ向ける。しっぽは bbox 外にはみ出してよい（bbox は本体領域のみ）。ナレーション（caption）・効果音（handwritten）・機械音声など話者のいないフキダシはしっぽを付けない。しっぽとフキダシ配置（bbox/anchor）と発言順（＝立ち位置）が三者一貫して話者を指すよう揃える。
   - **長いセリフは、フキダシ1つに無理なく収まる長さに分割する。** 句点・読点・意味の切れ目で分けて、同じ `bubbles` 配列内に別の `text` 要素として並べる（話者が同じなら2つ目以降の `speaker` は省略）。目安は1フキダシあたり20〜40文字程度、縦書きで2〜3行に収まる分量。1つの `text` が長すぎるとフキダシが巨大化して画面バランスを崩したり、文字が潰れて読めなくなる。

8. **フキダシの形は、フキダシごとに `shape` で指定する。**
   - フキダシの形は種類ごとに固定しない。ネーム担当がセリフの感情・種類に合わせて、その都度適切な形を選ぶ。
   - 語彙は次の9種：
     | shape | 形 | 向く用途 |
     |---|---|---|
     | `normal` | 丸い通常フキダシ | 通常の会話（**省略時のデフォルト**） |
     | `thought` | もくもく雲形 | 心の声・想像・夢見 |
     | `square` | 角形の枠 | 内心の独白・回想の語り・クールな内語 |
     | `caption` | 枠外キャプション箱 | ナレーション・時間経過・場所説明（コマの隅） |
     | `flash` | ギザギザ | 叫び・強い驚き・電話や機械の音声 |
     | `uniflash` | ウニフラッシュ（無数の針状の線が放射状に伸びるトゲトゲ） | 最大級の絶叫・衝撃の極致・ドンデン返しの「なっ…！？」・**心の中の絶叫ツッコミ**（monologue併用可） |
     | `wobbly` | よれよれ波線 | 弱り声・恐怖・ぼそぼそ・脱力 |
     | `whisper` | 点線 | ひそひそ声・内緒話 |
     | `handwritten` | フキダシなし手描き文字 | 効果音・小さなつぶやき・ゆらぎ文字 |
   - 迷ったら：叫んでいれば `flash`、**そのページ最大の絶叫・衝撃なら `uniflash`**（1ページ1個までが目安——多用すると効かなくなる）、心の中なら `thought` か `square`（キャラの性格で選ぶ——感情的なら雲、理性的なら角）、風で震える声・か細い声は `wobbly`、地の文は `caption`。
   - **ギャグの内心絶叫ツッコミは `monologue: true` ＋ `shape: "uniflash"` の組み合わせでよい**（例:「それは本気で言ってるのか！？」）。声には出ていないがウニフラで炸裂する、ギャグ漫画の定番演出として許可。この場合もウニフラの1ページ1個目安は守る。

9. **心の声（括弧付きセリフ）のフキダシには `monologue: true` を付け、`text` からは括弧「（）」を取り除く。**
   - `monologue` は**フキダシ単位**のフラグ。声に出したセリフと心の声が同じコマに同居してよい（心の声のフキダシにだけ付ける）。
   - 心の声であることの意味付けは `monologue` フラグ、**見た目は `shape`** の仕事（`thought`・`square` のほか、驚愕の内心ツッコミなら `uniflash`・`flash` も自由に選ぶ）。
   - 括弧文字を `text` に残すと、生成画像のフキダシの中にまで「（）」が文字として描かれてしまう。原稿の括弧は「心の声の目印」として読み取ったら捨てる（例: 原稿「（…！？）」→ `text: "…！？"` ＋ `monologue: true` ＋ `shape: "thought"`）。

10. **表情・動作は `action` に、セリフとは分離して書く。**
   演出指示（フキダシ内の文字を大きくする等）は `action` に書かず、該当フキダシの `emphasis` に書く。

11. **キャラクター名は全編で統一する。**（表記ゆれを見つけたら統一し、その旨を報告する）

12. **固定フィールドを必ず含め、§B・§C の内容を改変せずそのまま埋め込む。**
   `meta` の書式指定（右綴じ・縦書き・アンチック体）、ページヘッダー（左上タイトル・右上ページ番号）、ページフッター（下端中央の作者名。`meta.author` があるときのみ）、`style_notes`（§Bの配列をそのまま）、`layout_spec`（§Cのテキストをそのまま）。これらは画像生成AIに漫画としての約束事を伝えるためのもので、原稿が変わっても内容は固定。`layout_spec` は特に、**Nページ原稿を必ずN枚の別画像として出力させる**ための最重要ブロックを含む。

13. **OMNYの基盤であるYAMLの落とし穴に注意する。**
   同じマップ内に同じキーを2回書くと、OMNYの基盤であるYAMLは後の値で前を上書きし、片方が黙って消える。セリフをコマ直下の個別キーで書かず、必ず `bubbles` 配列に統一するのはこのため。

14. 出力後、**使用した画像アセットのリスト**（materials の一覧と各コマとの対応）を添え、続けて変更点・判断した箇所（コマの寄せ方、コマ割りの意図、bg配分の意図、表記統一など）を簡潔に添える。

## §A Nyamuru Data Model（NDM）v10 スキーマ定義（OMNY表現）

```
下記のNyamuru Data Model（NDM）v10をOpen Manga Name YAML（OMNY）形式で記述した内容に従って漫画を生成して。 #ChatGPT向け冒頭プロンプト。スキーマ違反でもこれも含める。
manga:
  schema_name: "Nyamuru Data Model"   # NDMの正式な識別名・固定。絵文字は含めない
  schema_version: 10                  # NDMのバージョン・整数
  meta:
    title:                # 作品タイトル
    author:               # 作者名（任意。指定時は各ページ下端中央のフッターに描く）
    format:               # 例: カラー漫画
    page_count:           # 総ページ数（整数）※ pages 配列の要素数と一致
    reading_direction:    # "right-to-left"（右綴じ・固定）
    text_orientation:     # "vertical"（縦書き・固定）
    font:                 # "アンチック体"（固定）
    style_notes:          # 作画上の共通指示（配列・§Bの内容を貼る）

  layout_spec:            # コマ割り座標系の説明文（画像生成AI向け・§Cの内容を貼る）

  setting:
    location:             # 場所
    time:                 # 時間帯
    weather:              # 天候
    background:           # 背景要素（配列）
    props:                # 小道具（配列）
    situation:            # シチュエーション説明

  characters:             # 登場人物（配列）
    - name:               # 名前
      role:               # 役割・キャラ属性

  materials:              # 素材一覧（作画エージェントの収集リスト）
    - key:                # コマの assets から参照する素材名（例: "キャラAの普段着"）
      type:               # "character" / "background" / "prop"
      keywords:           # ファイル名検索の手がかり（配列。例: ["キャラA", "普段着", "立ち絵"]）
      note:               # 見た目の要点＝素材が見つからない場合のデザイン指針（任意）

  pages:                  # ページ（配列）
    - page:               # ページ番号（整数）
      canvas:             # このページのキャンバス（原則 w:210, h:297, unit:mm）
        w: 210
        h: 297
        unit: mm
      panels:             # コマ（配列・並び順=読み順）
        - id:             # コマ番号（1始まり、右上→左下）
          shape:          # コマの形状
            type:         # "rect" または "polygon"
            # rect の場合: x, y, w, h（x,yは左上座標・mm）
            # polygon の場合: points [[x1,y1],[x2,y2],...]（頂点座標・mm）
          bg:             # 背景詳細度 0/1/2（必須）0=感情色＋効果のみ 1=簡易背景 2=詳細背景(1ページ2コマまで・位置自由)
          figures:        # コマ内の人物配置（配列・任意。無指定コマは発言順規則で自由構図）
            - name:       # キャラ名（characters と一致させる）
              bbox:       # {x, y, w, h} キャンバス絶対座標(mm)・描画領域の目安
              anchor:     # right/left/center/top-right/top-left/bottom-right/bottom-left（厳守指定）
              size:       # full/waist-up/bust-up/face（bbox寸法より優先）
          assets:         # このコマで使う素材（materials の key の配列・任意）
          bubbles:        # フキダシ（配列・縦書きなので右上のフキダシから順に）
            - text:       # フキダシの本文（必須）
              speaker:    # 話者名（任意。省略=同コマ内の直前のフキダシと同じ話者。ナレーション・効果音は省略）
              shape:      # フキダシの形（任意。normal/thought/square/caption/flash/uniflash/wobbly/whisper/handwritten。省略=normal）
              emphasis:   # そのフキダシ内で大きくする語（任意）
              monologue:  # このフキダシが心の声なら true（任意。見た目は shape で指定）
              bbox:       # {x, y, w, h} フキダシ本体の目安(mm)。しっぽは含まない
              anchor:     # コマ内相対位置（厳守指定。top-rightは最初に読まれるフキダシ）
          action:         # 表情・動作のト書き（無ければ null）
```

### フィールドの決まりごと

- `meta.page_count` は整数、`pages` は実際のページを並べた配列。成形後、`page_count` は `pages` の要素数と一致させる。
- `meta.author` は任意。原稿・依頼に作者名があれば入れる（勝手に創作しない）。指定時は各ページ下端中央のフッターとして作画される（座標は§C、作画の扱いは§Bを参照）。
- `meta.reading_direction` / `text_orientation` / `font` は固定値（`"right-to-left"` / `"vertical"` / `"アンチック体"`）。
- `meta.style_notes` には §B の内容を、`layout_spec` には §C の内容を、それぞれ改変せず埋め込む。`layout_spec` は多行文字列（`|` ブロック記法）。
- `materials` は**この作品で使う素材のカタログ**。`key` は日本語の分かりやすい名前、`keywords` はファイル名検索の手がかり。作画エージェントは作業前にこれで素材を収集する。素材が無い場合は `note` を指針に自由デザイン。
- `pages.canvas` はA4基準。原則として全ページ同じ（`w:210, h:297, unit:mm`）。
- `pages.panels` の配列順序＝読み順。並び順と `id` を一致させる。
- `pages.panels` のコマ数は**1ページ最大6**。7コマ以上はセリフの描画が崩れるため禁止（手順2参照）。
- `panels.shape.type` は `"rect"` か `"polygon"` のいずれか。
- `panels.bg` は **0/1/2 の整数・全コマ必須**。0=背景なし（感情色ベタ/グラデ＋効果のみ）、1=簡易背景（シーン色＋記号的要素1〜2点）、2=詳細背景（**1ページに2コマまで**・位置は自由）。
- `panels.assets` は任意。そのコマで参照すべき素材の `key` を並べる。bg:0の感情コマなど素材不要のコマは省略。
- `bubbles` は**常に配列**（フキダシ1個でも配列）。`text` は各フキダシの必須フィールド。長いセリフの分割は手順7に従う。
- `bubbles[].speaker` は任意。省略時は**同じコマ内の直前のフキダシと同じ話者**。ナレーション（caption）・効果音（handwritten）など話者のいないフキダシも省略する。
- `bubbles[].shape` は任意。9種の語彙（normal/thought/square/caption/flash/uniflash/wobbly/whisper/handwritten）から選ぶ。省略時は `normal`（丸い通常フキダシ）。`uniflash` は1ページ1個までが目安。
- `bubbles[].emphasis` は任意。フキダシ内で強調したい語だけ指定。
- `bubbles[].monologue: true` は心の声のフキダシにのみ付ける。原稿で括弧「（）」付きだったセリフが対象だが、`text` には括弧を残さない（手順9参照）。**見た目の枠形状は `shape` で指定する**（四角固定ではない）。
- `action` は表情・動作のみ。無い場合は `null`。

## §B style_notes の内容（作画・演出ルール一般）

`meta.style_notes` にはこの配列をそのまま使う。作画・演出に関するルールはここに一本化してある。
※ bg の定義やマージン等の数値は「手順」「§C」にも登場する（ネーム担当向けと画像生成AI向けで読者が違うための意図的な二重掲載）。**ルールを改定するときは必ず両方を揃えて直す**こと。

```yaml
style_notes:
  - "セリフ、フキダシ、モノローグ、作品タイトル、ページ番号を、作画と同時に画像生成AIで描画する"
  - "文字のない原稿を生成して、後からプログラム・画像編集・ローカル処理で写植することは禁止"
  - "各ページを、セリフとフキダシを含む完成漫画画像として一度に生成する"
  - "セリフはOMNY記載どおりの日本語を使用し、縦書きで描画する"
  - "文字化け・誤字・欠落・フキダシ外へのはみ出しがある場合は、後加工せず、そのページだけ画像生成で再生成する"
  - "生成された完成PNGを加工せず、そのまま納品する"
  - "作画を始める前に、materials の keywords を手がかりに手元の素材ファイル（立ち絵・背景・小物）を収集し、各コマの assets に挙がっている素材をリファレンスとして参照して描く"
  - "assets に対応する画像素材が見つからない場合は、素材待ちで止まらず、materials の note と作品のスタイルに合った見た目をその場で自由にデザインしてよい。一度デザインした見た目は以後のコマでも一貫して維持する"
  - "背景素材は『そのシーンの美術設定資料』として参照する——場所のデザイン・色彩・光・小物の配置を踏襲するためのものであり、素材画像の構図をそのまま複写してキャラクターを上に置く合成は禁止。コマごとにカメラアングル・画角・構図は自由に変えて（寄り・引き・あおり・俯瞰・キャラの背後から等、そのコマの演出に合う視点で）背景を描き起こし、キャラクターと背景を同じ光源・色温度・影・遮蔽の前後関係で一体の絵として描いて馴染ませる。キャラの足元には接地影を必ず入れる"
  - "ト書き（表情・動作）はセリフと分離。actionはキャラクターの表情・動作のト書きで、セリフとは独立して描画に反映する"
  - "各フキダシの形は shape の指定に従って描く：normal=丸い通常フキダシ／thought=もくもく雲形（心の声・想像）／square=角形の枠（内心の独白・回想の語り）／caption=枠外キャプション箱（ナレーション・時間経過、コマの隅に角箱）／flash=ギザギザ（叫び・強い驚き・電話や機械の音声）／uniflash=ウニフラッシュ＝無数の細い針状の線が放射状に伸びるトゲトゲのフキダシ（最大級の絶叫・衝撃の極致。monologue: true との併用＝声に出ていない心の中の絶叫ツッコミにも使ってよい）／wobbly=よれよれ波線（弱り声・恐怖・ぼそぼそ）／whisper=点線（ひそひそ声）／handwritten=フキダシなしの手描き文字。shape の指定が無いフキダシは normal（丸）で描く"
  - "monologue: true のフキダシは心の声（フキダシ単位のフラグ。同じコマに、声に出したセリフと心の声のフキダシが同居することがある）。フキダシの中に括弧「（）」の文字は描かない（心の声と分かる見た目は shape の指定に任せる）"
  - "各コマの bg は背景詳細度。bg:0=具体的背景なし。シーン・感情に合った背景カラー（ベタ塗り or グラデーション。怒り=赤系/喜び=暖色/驚き=黄系/緊張=紫〜紺/感動=金〜桃/静寂=青系など）を敷き、その上に集中線・漫符・トーン・オーラ・キラキラ等の効果を乗せて、キャラの表情と感情に視線を集中させる。bg:1=簡易背景。シーンに合った背景色グラデーションの上に、その場を示す記号的な要素を1〜2点（壁の線・窓枠・机の縁・手前の小物のシルエット等）。手前オブジェクト＋奥は色グラデの奥行き構成も良い。bg:2=詳細背景。ロケーションをしっかり描き込む見せゴマ"
  - "bg:0 のコマに風景・室内・小物などの具体的背景を描き込んではいけない（感情色ベタ/グラデ＋効果表現のみ）。bg:0/1 で白一色の背景は禁止——必ずシーンに合った色を敷く。bg の指定はコマごとに厳密に守る"
  - "bg:0/1 の背景カラーは低〜中彩度（くすみ・パステル寄り）に抑える。背景グラデは雰囲気を伝えるための色であり、キャラとセリフより主張してはいけない。高彩度の鮮やかな背景色は、感情の爆発や最大の強調など明確な演出意図があるコマだけに限定する"
  - "bubblesは配列。上から順に読む。一つのコマに複数のフキダシを含めてよい。各フキダシは**その話者のすぐ近く（頭の上・顔の横など）に配置し、コマ内のフキダシの並びは話者の並び順に合わせる**。話者は発言順に右から左へ並ぶため、フキダシも自然に右上→左下の読み順になる。話者のいないフキダシ（caption・handwritten・機械音声など）はコマの隅や空きスペースに置く"
  - "各フキダシの話者は speaker で示す。speaker が省略されたフキダシは、同じコマ内の直前のフキダシと同じ話者（ナレーション・効果音は話者なし）。**フキダシには尻尾（しっぽ）を付けて話者を指す**——normal/thought/flash など通常発話系のフキダシは、しっぽの先端を speaker のキャラの口元付近へ向ける。しっぽはフキダシ本体（bbox）の外にはみ出してよい。caption・handwritten・機械音声など話者のいないフキダシにはしっぽを付けない。しっぽに加えて、フキダシをその話者の近く（頭の上・顔の横など）に配置することで対応をより明確にする"
  - "コマ内のキャラクターの立ち位置は発言順と連動させる——そのコマで最初に発言するキャラ（bubbles 配列の最初の話者）をコマの右側に置き、以降の話者は発言順に右から左へ並べる。フキダシも各話者の近くに置かれるため、フキダシの読み順とキャラの並びが自然に一致する（ネームに立ち位置の指定は無いので、発言順を配置の指針として使う。actionで構図・立ち位置が明示されているコマはそちらを優先）"
  - "bubbleのemphasisは、そのフキダシ内で大きな文字にする語。同じフキダシ内で1.3〜1.5倍のサイズで描く"
  - "セリフは全て縦書き（文字は上から下、列は右から左へ送る）"
  - "本文フォントはアンチック体（漢字=明朝体、かな=ゴシック体の混植）。心の声・キャプションの枠内も同じくアンチック体"
  - "ページは右綴じ、コマは右上→左下の順（id 番号の昇順）に読む"
  - "各ページの左上に作品タイトル、右上に『現在/総ページ数』のページ番号を配置する。ヘッダーはコマ本体に被らない、文字色は黒、フキダシは付けない(地の紙面に直接描く)"
  - "meta.author に作者名がある場合、各ページの下端中央に作者名を小さく配置する。文字色は黒、フキダシは付けない(地の紙面に直接描く)。下端まで断ち切られたタチキリコマと重なる場合はコマ絵の最前面に重ね、読めるよう必要に応じて文字に細い白フチを付ける。author が空・未指定ならフッターは描かない"
  - "キメ台詞・決意表明・啖呵・重要リアクションでは、話者の顔をアップに描く(バストアップ〜顔面クローズアップ)。コマサイズも前後より一回り大きく取り、余白・集中線・グラデーション背景等でセリフの重みを引き立てる。該当コマの action に「顔のアップ」「バストアップ」「決意の表情」等があればそれを優先する"
  - "長いセリフはフキダシに収まる長さに分割して複数フキダシにする"
  - "効果線・集中線は action や場面から適切に判断して追加してよい"
  - "作品全体の背景色は、映像のカラースクリプトのように感情の起伏に沿った色の流れ（ドラマカーブ）として繋がりを意識する。隣接コマの色相差が大きいほど場面転換・感情転換が強く伝わる。照明表現（逆光=シリアス、トップライト=神聖、下から=不穏、光のボケ玉=感動など）も漫符・グラデ選択のヒントにしてよい"
  - "figures・bubbles の bbox/anchor/size の解釈は layout_spec の『人物・フキダシのバウンディングボックスの解釈』に従う。優先順位は action の構図指示＞anchor（厳守）＞bbox（目安）"
```

## §C layout_spec の内容（画像生成AI向け・座標/寸法/空間仕様）

`layout_spec` フィールドにこのテキストを **そのまま** ブロック記法で埋め込む（改変しない）。作画・演出ルール一般は `style_notes`（§B）に集約してある。

```
# コマ割り座標系の説明（画像生成AI向け）

## 出力枚数(最重要)
- Nページの原稿は必ずN枚の別画像として出力する
- 1枚の画像内に複数ページをまとめてはいけない(見開き扱いも禁止)
- 各画像はA4縦(幅210mm × 高297mm)1ページ分のみ
- 出力は page1, page2, ... と1ページずつ独立した画像で
- ページ間の余白・並列表示・サムネイル一覧・見開きスプレッドといった発想は全て禁止

## キャンバス
- サイズ: A4(幅210mm × 高297mm、縦長)
- 原点: 左上を(0,0)。右方向にx増加、下方向にy増加。単位はmm。

## コマ形状
- shape.type "rect": x, y, w, h で四角コマ(x,yは左上座標)
- shape.type "polygon": points [[x1,y1],[x2,y2],...] で頂点座標(多角形・対角線カット用)

## コマ間の間隙とライン
- 各コマの外周に約2〜3mmの白マージンを取る
- コマ枠線は太さ約1mm、黒。隣接コマの枠線は共有せず独立して描く
- 例外: タチキリ(断ち切り)コマ。座標が x=0 / x=210 / y=297 に達しているコマは、その辺を紙端まで余白なしで描き切る(断ち切り辺に枠線・白マージンを入れない)。他の辺は通常どおりマージンと枠線を描く。上端はヘッダー帯があるため y=10 が上限

## 背景詳細度(bg)
- 各コマの bg 値(0/1/2)は背景の描き込み量の指定。詳細は style_notes を参照
- bg:0 のコマは背景描写ゼロ(感情色＋効果のみ)。bg:2 のコマは1ページに2コマまで(位置は自由)

## 素材(assets)
- 各コマの assets はそのコマで参照する素材(materials の key)の一覧
- 素材の収集方法・見つからない場合の扱いは style_notes を参照

## ページヘッダーの座標
- ヘッダー要素は上端の y=0〜10mm の帯(コマの外側)に配置する
- 左上要素(作品タイトル): 座標目安 x=10, y=7、左揃え、フォントサイズ約5mm
- 右上要素(ページ番号「現在/総ページ数」形式、例 "3/12"): 座標目安 x=200, y=7、右揃え、フォントサイズ約5mm
- ヘッダーはコマ本体(y=10 以降)に被らない
- 表示内容と作画上の扱い(フォント種・色・フキダシ有無など)は style_notes の指示に従う

## ページフッターの座標
- フッター要素は下端の y=287〜297mm の帯に配置する
- 中央要素(作者名 meta.author): 座標目安 x=105, y=294、中央揃え、フォントサイズ約4mm
- meta.author が空・未指定の場合はフッターを描かない
- 下端タチキリコマ(y=297 まで達するコマ)と重なる場合もフッターは最前面に重ねて描く(作画上の扱いは style_notes の指示に従う)

## 全体トーン
- カラー漫画。線画＋ベタ塗り、必要に応じてグラデーション

## 作画・演出ルール
- 読み順、セリフ・フキダシの扱い(形状shapeを含む)、モノローグ、表情・動作、背景詳細度、素材、見せ場の演出、効果線などのルールは meta.style_notes を参照する

## 人物・フキダシのバウンディングボックス（figures / bubbles.bbox）の解釈

### 基本原則: 三層の優先順位
配置指定は次の優先順位で解釈する。上位と下位が矛盾する場合は常に上位に従う。
1. action の構図指示（「顔のアップ」「背後から」等の明示的な演出指定）が最優先
2. anchor（相対位置の語彙）— 必ず守るべき配置命令
3. bbox（mm座標）— 目標となる位置・大きさの目安。数mmの誤差は許容

bbox をピクセル単位で厳密に再現しようとしてはならない。bbox が伝えるのは
「コマ内のどのゾーンに・どのくらいの大きさで」であり、構図の自然さ・
キャラのポーズ・パースの整合が bbox の厳密な再現より優先される。

### figures（人物配置）
- name: そのコマに描くキャラクター名。figures に列挙されたキャラは必ず全員描く。
  figures に無いキャラをコマ内に描いてはならない（action でコマ外の声と
  指定されたキャラは、フキダシのみでよい＝姿を描かない）。
- bbox: {x, y, w, h} キャンバス絶対座標(mm)。キャラの描画領域の目安。
  - 縦横比から全身/バストアップの見当をつけ、size の指定と併せて解釈する
  - キャラの頭部は bbox の上部1/3以内に収まるように描く（顔位置の予測可能性
    のため。フキダシ回避判定がこの仮定に依存している）
- anchor: コマ内の相対位置。right / left / center / top-right / top-left /
  bottom-right / bottom-left の語彙。この指定は厳守する。
  同一コマに複数キャラがいる場合、anchor の相対関係（誰が右で誰が左か）を
  入れ替えてはならない。
- size: 描画サイズの語彙。full（全身）/ waist-up（腰から上）/
  bust-up（胸から上）/ face（顔のクローズアップ）。bbox の寸法より優先する。

### bubbles の bbox / anchor
- bbox: フキダシの位置と大きさの目安(mm)。縦書きテキストが無理なく入る
  サイズに調整してよいが、大きさの序列（大きい bbox のフキダシほど大きく
  描く）は維持する。
- anchor: コマ内の相対位置。厳守する。特に top-right（コマ右上）は
  「そのコマで最初に読まれるフキダシ」を意味するため、他のフキダシより
  右上に配置されなければ読み順が壊れる。
- フキダシ同士の読み順は bubbles 配列の順序＝右上→左下。bbox の座標と
  配列順が食い違って見える場合は配列順を優先し、右上→左下が成立する
  位置に補正する。

### 禁止事項（bbox 版の追加ルール)
- フキダシを、いかなるキャラの顔（頭部）にも重ねてはならない。
  figures の bbox 上部1/3 は顔ゾーンとして予約されている。
- フキダシがキャラの胴体・背景に重なるのは許容される（通常の漫画表現）。
- しっぽ（フキダシの尾)は speaker のキャラに向ける。bbox はしっぽを
  含まない本体領域なので、しっぽは bbox の外にはみ出してよい。
- polygon コマでは、bbox がコマの斜辺からはみ出す場合、コマ内に収まる
  よう内側に寄せて描く（斜辺を跨いで描画しない）。

### フォールバック
- figures が無いコマ（bg:0の感情コマ等）は、発言順規則
  （発言順＝立ち位置、最初の話者が右）で自由に構図を決めてよい。
- bbox と anchor が矛盾している場合（例: anchor:"left" なのに bbox が
  コマ右半分）は anchor に従い、bbox は大きさの目安としてのみ使う。
```
