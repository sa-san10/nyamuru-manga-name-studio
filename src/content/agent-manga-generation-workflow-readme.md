Open Manga Name YAML（OMNY）形式の漫画ネームとリファレンス画像を使い、AIエージェントが漫画生成から検品・納品までを一貫して進めるためのワークフローです。

OMNYの解析、必要素材の準備、ページ単位の画像生成、失敗ページの再生成、ファイル命名、検品レポート、最終報告までの手順を定義しています。

本ワークフローはChatGPTWorkを想定しています。

## 用意するもの

- `agent-manga-generation-workflow.md`：エージェントが参照するワークフロー本体
- OMNY形式の漫画ネームファイル
- キャラクター、背景、小道具などのリファレンス画像
- 画像生成とファイル操作が可能なAIエージェント環境

リファレンス画像には、対応するキャラクターや背景が判別できるファイル名を付けてください。必要に応じて、簡単な設定メモを同じ作業フォルダへ配置すると参照の精度が上がります。

## 使い方

1. ワークフロー本体を、AIエージェントが参照できる場所へ配置します。
2. 下記の「プロジェクト指示例」をエージェントのプロジェクト指示へ追加します。
3. OMNYファイルと必要なリファレンス画像を渡します。
4. エージェントに漫画生成を依頼します。

## プロジェクト指示例

```text
# このプロジェクトでは漫画を制作します

## 作業手順

1. `agent-manga-generation-workflow.md` を読み、内容を理解する。
2. ユーザーから渡されたOMNY形式の漫画ネームを読み込む。
3. 必要なキャラクター立ち絵や背景素材を検索し、使用する分だけ作業場所へ複製する。素材の識別ができなくなるため、ファイル名は変更しない。
4. 準備が完了したら、ワークフローとOMNYの指定に従って漫画生成を開始する。
5. 修正や再出力でも、ユーザーの明示指示がない限りコマ割りを変更しない。
6. ユーザーから渡されたOMNY原本を、内容を変えずファイルとして成果物に含める。
7. ワークフローに従い、検品・結果報告・納品まで進める。
```

## 入力と成果物

| 種類 | 内容 |
|---|---|
| 入力 | OMNY形式の漫画ネーム、リファレンス画像、必要に応じた追加指示 |
| 成果物 | ページ単位の生成画像、OMNY原本、検品レポート、作業結果の報告 |

具体的な生成ルール、再生成条件、命名規則、検品項目はワークフロー本体を参照してください。
