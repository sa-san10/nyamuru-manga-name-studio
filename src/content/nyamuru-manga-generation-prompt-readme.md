ストーリー原稿やアイデアを、**Nyamuru Data Model（NDM）v10** に準拠した漫画ネームへ構造化し、**Open Manga Name YAML（OMNY）形式**で出力するためのLLM向けプロンプトです。

台本、あらすじ、テーマなどを渡すと、ページ構成・コマ割り・人物配置・フキダシ・背景・素材参照を含む、機械可読な漫画ネームを生成します。

縦書き右閉じ漫画用のプロンプトです。
他の漫画形式にする場合は、連動変更箇所が多いため、LLMで編集することをお勧めします。

## このプロンプトでできること

- 原稿をページとコマへ分解
- A4絶対座標によるコマ割りを設計
- 人物とフキダシの位置・大きさを指定
- セリフ、モノローグ、効果音、演出を構造化
- 背景詳細度とリファレンス素材を指定
- NDM v10のルールに沿ってOMNYを自己検証

## NDMとOMNY

**Nyamuru Data Model（NDM）** は、漫画ネームの構造、コマ割り、人物、フキダシ、素材参照を定義するデータモデルです。愛称は **Nyamuru🐱** です。

**Open Manga Name YAML（OMNY）** は、NDMをYAMLとして保存・交換するためのシリアライズ形式です。

## 用意するもの

- `nyamuru-manga-generation-prompt-v10.md`：LLMへ渡す完全版プロンプト
- 漫画化する台本、あらすじ、テーマ、セリフなど
- 長いプロンプトと原稿を一度に扱えるLLM環境

本プロンプトはClaude Fable 5以上のLLMを想定しています。

## 使い方

1. 完全版プロンプトを、LLMのプロジェクト指示またはシステム指示へ設定します。
2. 漫画化したい原稿と、ページ数や作風などの希望を伝えます。
3. LLMが出力したOMNYを保存し、内容と検証結果を確認します。
4. 必要に応じてOMNYを編集し、漫画制作ワークフローへ渡します。

## 依頼例

```text
次の原稿を、Nyamuru Data Model v10に準拠した漫画ネームへ変換してください。
出力はOpen Manga Name YAML（OMNY）形式にしてください。

ページ数：4ページ
形式：カラー漫画
希望する雰囲気：明るい日常コメディ

原稿：
（ここに台本やあらすじを入力）
```

## 入力と出力

| 種類 | 内容 |
|---|---|
| 入力 | 台本、あらすじ、テーマ、希望ページ数、作風や演出の指定 |
| 出力 | NDM v10に準拠したOMNY形式の漫画ネーム、検証結果 |

生成結果は必ず内容を確認してください。スキーマの詳しい構造や各フィールドの意味は、Nyamuru Data Modelの仕様を参照してください。
